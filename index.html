Neural Networks for iOS

Final Year project on convolutional neural networks for iOS gesture recognition. Initial training was done using the MNIST dataset. Currently recording my own dataset of iOS gestures which will allow training of more accurate networks for iOS. The better accuracy come from extra paramaters such as the time, speed, acceleration and force with which the numbers are drawn. This information allows the networks to predict what the number is more accuratly than is possible with standard bitmap images. This dataset will be open sourced after completion to allow for further work in this area.

The app for recording this data was created in Xcode using Swift. Initally it asks for the users age, sex, nationality and handedness. This is to ensure that the dataset is diverse. The database was built using Apple's CoreData framework. It consists of four levels. The first level is the user. Each user has 80 digits associated with them. These 80 digits are comprised of 8 sets of the numbers 0 to 9. Four of these sets are drawn using the index finger and the other four are drawn using the thumb. Each of these digits are comprised of a variable number of strokes. E.g. the number 7 can be drawn in 1 or 2 strokes. Each of these strokes are made up of a number of x,y co-ordinates.

Deep Convolutional Neural Networks

localhost

Convolutional neural networks read in the input images in batches and pass a filter over the pixels of each image. The convolution operation is then performed on the pixels exposed by the filter. The convolution operation allows features to be extracted from the images. Down sampling then occurs in order to reduce the computational power requred to process the data. These steps are then repeated and each iteration is known as a layer in the network. Each consecutive layer results in more and more abstract feature extraction. After completing the iterations the output is then classified using softmax based on the features that have been extracted. This means that the values for each feature in the final layer are converted to a percentage. This percentage gives how likely the input is to match each of the possible classifications.
